Spring Boot 整合 Activiti7

NO：2020-0x01

前言

  这段时间在公司项目的需求下我开始了折腾流程相关的业务开发，前段时间我就写了有关于 Bpmn.js 流程编辑器跟 Vue 的整合以及汉化的文章。因为流程前端这部分我负责搞，涉及到的流程相关的，流程在线编辑，流程列表模块，流程运行图等。然后后端模块，Spring Boot2.x 和 Activiti7 流程引擎相关的，emmmm 没错，还是我在搞。哈哈哈哈哈哈哈哈哈。所以这次我来开坑了，我决定做一个 Activiti7 的系列，从 Spring Boot 整合 Activiti7 到流程部署，任务办理等系列的文章。

  我在 GitHub 中开了一个仓库专门用来存放这个系列的所有 demo 以及每篇的一个最终代码。对于每个章节我会给定一个编码然后创建一个分支存放该章节的代码，master 分支就是所有章节的一个最新完整 demo。比如 2020-0x01 分支将对应的就是当前章节的一个 demo 代码了。

Github 完整代码 Activiti-seed 作者博客 Winily-blog

  在开始做这个流程之前先来看看百度的简介稍微来了解一下什么是流程引擎吧。

  所谓工作流引擎是指 workflow 作为应用系统的一部分，并为之提供对各应用系统有决定作用的根据角色、分工和条件的不同决定信息传递路由、内容等级等核心解决方案。工作流引擎包括流程的节点管理、流向管理、流程样例管理等重要功能。

  大概大概就是这么一个意思。Activiti 呢就是众多开源流程引擎中的一个。Activiti7 就是这个引擎当前最新的版本了。
我写这个系列将会参照activiti7 官方文档来折腾，按照文档提供的最新版来安装依赖等操作，当然你不想看这个官方的文档也没关系，我会把有需要的代码或者步骤在文章中列出或者说明。咱要学就学最新的，但是实际开发还是要看需求。接下来我们就正式开搞。

准备开发环境，安装依赖

  开发的第一步肯定是先准备好开发环境，我们这次要做的事情是 Spring Boot 2.x 跟 Activiti7 的整合，所以一般来说来看这篇文章的都会有相关的 Java 以及 Spring 家族相关的基础了，所以我就不过多的来叙述这一部分了。接下来我大致列出开发所需的工具以及依赖。

  1. idea
  2. MySql
  3. 数据库连接器（我用的是 Navicat Premium）
  4. 流程编辑器（我这里直接使用了我前两篇文章中搭建出来的 bpmn.js）

以上就是所需的开发环境。在这次的过程中为了更加直观的展示和操作，做到了某一个步骤我会同步的开发出对应的界面，我将会在bpmn.js-demo中开启一个分支(process-center-demo)来存放相关的前端代码。接下来是导入依赖。

首先先创建一个 Spring Boot 的项目，创建的时候我们需要把一下三个依赖勾选上

  1. spring-boot-starter-web
  2. mysql-connector-java
  3. lombok

等创建完成后再来导入 Activiti、swagger 以及 MyBatis-plus 的依赖，将一下依赖复制到你的 pom 文件中, 在此为了方便接口的调试我加入了 Swagger2 的依赖来生成接口文档。

需要安装的依赖就这些了，需要注意的是，我这里使用的是 MyBatis-plus，在 Activiti7 中是内置了 MyBatis 的所以在这里必须将 Activiti 中的 Mybatis 依赖排除掉不然会发生错误。

注意踩坑：
  在 Activiti7 最新的版本中有一个很怪异的坑，在 7.1.0.M4 以上版本中自动创建数据表 ACTRE_DEPLOYMENT 表会缺少 VERSION 字段和 PROJECTRELEASE_VERSION 字段，要解决也很简单，直接自己手动去数据库给这个表加上这两个字段即可，类型都是 int(11)。
